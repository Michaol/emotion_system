# Bug Report: Emotion Engine 2.0.0 — State File Path Mismatch [FINISHED]

**Date:** 2026-03-20 08:24 UTC+8
**Status:** ✅ FIXED
**Fixed at:** 2026-03-20
**Package:** emotion-engine 2.0.0
**Component:** Python wrapper (`engine.py`)
**Severity:** High — decay results silently lost

---

## 问题描述

`get_engine()` 自动给状态文件名加 `_state.json` 后缀，导致 Python wrapper 和 MCP server 读写不同文件，衰减结果无法持久化。

## 受影响文件

`python/emotion_engine/engine.py` 第 392 行

## 代码位置

```python
# 当前代码 (第 390-393 行)
state_path = os.path.join(sdir, f"{aid}_state.json")
_instances[aid] = EngineWrapper(aid, cdir, state_path)
```

## 问题

- `get_engine(agent_id="asuna", state_dir=".../state")` → 读写 `asuna_state.json`
- MCP server 配置指向 → 读写 `asuna.json`
- 两个文件独立存在，数据不同步

## 复现

```python
from emotion_engine import get_engine
eng = get_engine(agent_id="asuna", state_dir="/path/to/state")
eng.get_state()   # 从 asuna_state.json 读取
eng.save()        # 写入 asuna_state.json
# MCP server 写的是 asuna.json，两边互不影响
```

## 期望行为

`get_engine()` 使用 `{aid}.json` 作为文件名，与 MCP server 保持一致。

## 修复建议

```python
# 改为
state_path = os.path.join(sdir, f"{aid}.json")
```

## 注意

- 如果已有用户使用 `asuna_state.json`，需要考虑迁移或兼容
- 这是 emotion_system 库自身问题，非 OpenClaw 集成问题
